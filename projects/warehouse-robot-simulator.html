Project Overview

This project addresses the inefficiencies in South African grocery retail warehouses by developing a virtual simulation of an AI-powered warehouse robot system. The simulator demonstrates how mobile robots can replace manual labor for order fulfillment using advanced AI techniques, particularly the Traveling Salesman Problem (TSP) optimization.

Problem Statement

With the rapid growth of e-commerce in South Africa, major retailers like Checkers Sixty60 and Pick n Pay ASAP have introduced same-day delivery services. However, these services still rely heavily on human workers manually retrieving items from shelves, following inefficient paths similar to regular shoppers. This results in:

  • Longer order processing times
  • Operational inefficiencies
  • Occasional delivery delays
  • Higher operational costs

Proposed Solution

The AI-powered warehouse robot simulator demonstrates how automated systems can optimize order fulfillment through:

  • TSP Optimization: Each robot acts as a "salesman" visiting item locations (cities) to find the shortest path
  • Multi-Agent Collaboration: Three intelligent agents work together to manage orders, optimize paths, and avoid obstacles
  • Real-time Adaptation: Dynamic response to changing warehouse conditions and obstacles
  • Load Balancing: Efficient distribution of orders across multiple robots

Intelligent Agent Architecture

Order Allocation Agent

Utility-Based Agent

Responsible for assigning customer orders to warehouse robots using a Round Robin load-balancing strategy. Ensures even distribution of workload and prevents bottlenecks.

  • Load balancing optimization
  • Real-time robot status monitoring
  • Efficient order queue management

Path Optimization Agent

Hybrid Agent (Goal + Utility)

Utilizes TSP solvers to calculate the shortest and most efficient path for each robot. Implements the Nearest Neighbor Algorithm while considering warehouse layout and congestion.

  • TSP-based path optimization
  • Dynamic route recalculation
  • Congestion avoidance

Obstacle Avoidance Agent

Goal-Based Agent

Enables robots to detect and avoid static obstacles (shelves, walls) and dynamic obstacles (other robots, people) using reinforcement learning strategies.

  • Real-time obstacle detection
  • Reinforcement learning navigation
  • Collision prevention

PEAS Framework Analysis

The system environment is analyzed using the PEAS (Performance, Environment, Actuators, Sensors) framework:

Agent Type Performance Measures Environment Actuators Sensors
Order Allocation Agent Efficient order distribution, minimizing idle time, balancing robot workload Virtual warehouse with multiple robots, orders, and checkouts Assigns orders to robots based on workload and availability System status updates on robot availability, order queue
Path Optimization Agent Shortest possible path, minimal energy use, quick order fulfillment Virtual warehouse with aisles, shelves, and pickup points Generates optimized paths for robots to follow Warehouse layout data, order item locations, robot positions
Obstacle Avoidance Agent Avoid collisions, smooth navigation Virtual warehouse with static and moving obstacles Adjusts robot movement to avoid collisions Lidar/camera-based object detection, real-time position tracking

Task Environment Properties

Partially Observable

Robots have limited sensor range and may not detect all obstacles instantly

Stochastic

Contains unpredictable elements like sudden movements of people or robots

Sequential

Current decisions impact future outcomes and robot interactions

Dynamic

Environment evolves in real-time with new orders and robot movements

Discrete & Continuous

Order allocation works in steps while obstacle avoidance is continuous

Multi-Agent

Multiple AI agents collaborate and influence each other's activities

Expected Benefits

  • Improved Efficiency: Robots complete orders faster than human workers
  • Cost Reduction: Reduced labor costs and operational expenses
  • Error Minimization: Automated systems reduce human errors
  • Scalability: Easy scaling for high-demand periods
  • Space Optimization: Better utilization of warehouse space
  • 24/7 Operation: Continuous operation without breaks

Technical Implementation

The simulator was developed using advanced AI techniques including:

  • Traveling Salesman Problem (TSP) Solvers: Nearest Neighbor Algorithm for path optimization
  • Reinforcement Learning: For dynamic obstacle avoidance and navigation
  • Multi-Agent Systems: Coordinated collaboration between different agent types
  • Load Balancing Algorithms: Round Robin strategy for order distribution
  • Real-time Simulation: Dynamic environment with moving obstacles and changing conditions
